

## Add "Download Setup" Button to Saved Setups

### What gets added
A **Download** button on each saved setup card that generates a printable PDF containing all the setup's data ‚Äî so users can print it out at the track or share it with their crew.

### Where the button lives
Inside the existing action row in each saved setup `CardHeader` (next to the Edit/Delete buttons at lines ~645‚Äì660 of `src/pages/Setups.tsx`). Same icon-button styling as siblings, using lucide's `Download` icon. Clicking it does NOT toggle the card expand/collapse (stop event propagation, like the other action buttons).

### What's in the PDF
A clean, printable one-page-ish layout with:

1. **Header** ‚Äî Setup name, created date, fastest lap (if set)
2. **Metadata block** ‚Äî Car (name + year/make/model), Event, Track, Session
3. **Chassis data** ‚Äî Only renders sections that have data:
   - Alignment (camber, caster, toe, ride height per corner) ‚Äî 4-corner grid
   - Springs & Shocks per corner
   - Weights & percentages (front/rear/left/right/cross)
   - Tire pressures (cold/hot per corner)
   - Tire temps (in/center/out per corner)
4. **Notes** ‚Äî `notes_times` field if present
5. **Attached files list** ‚Äî File names of any uploaded sheets (with note that they're stored in the app, not embedded ‚Äî keeps PDF small and avoids signed-URL fetch complexity)
6. **Tire wear photos** ‚Äî Listed by corner if present (same note as attachments)
7. **Footer** ‚Äî "Generated by Track Side Ops ‚Ä¢ [date]"

### Tech approach
Use **`jspdf`** (small, client-side, no server roundtrip, no extra deps already in tree). Reasons:
- Pure browser ‚Äî works offline, no edge function needed
- Already a common pick for this kind of print-friendly export
- Library is ~50KB gzipped

The PDF generator lives in a new file: **`src/lib/exportSetupPdf.ts`** ‚Äî keeps `Setups.tsx` from getting larger and makes the function reusable later (e.g. bulk export from a future "compare setups" view).

### Data fetching
Saved setup list currently only has summary fields (`id`, `setup_name`, `session_name`, `car_id`, `event_id`, `created_at`, `fastest_lap_time`, `notes_times`). The full `setup_data` row (all 50+ chassis fields) is needed for the PDF ‚Äî fetch it on-demand when the Download button is clicked, so we don't bloat the initial page load.

```ts
const { data: fullSetup } = await supabase
  .from("setup_data")
  .select("*")
  .eq("id", setup.id)
  .single();
```

### Files changed
- **`src/pages/Setups.tsx`** ‚Äî add Download icon-button to each saved setup's action row; wire up onClick handler that fetches full setup, calls the PDF generator, and triggers browser download
- **`src/lib/exportSetupPdf.ts`** *(new)* ‚Äî `exportSetupToPdf(setup, car, event, attachments, tirePhotos)` function that builds the PDF with jsPDF and calls `doc.save(filename)`
- **`package.json`** ‚Äî add `jspdf` dependency

### Visual treatment
- Button: same `variant="ghost" size="icon"` as Edit/Delete buttons already on the row
- Icon: `Download` from lucide-react, size 16
- Tooltip via `title` attr: "Download setup as PDF"
- Filename: `setup-{name-slugified}-{YYYY-MM-DD}.pdf`

### Out of scope
- Embedding tire wear photos or attached files inside the PDF (would require fetching binaries + base64 encoding ‚Äî possible later, but inflates file size and adds latency)
- Bulk download / multi-select export
- PDF customization options (paper size, theme, logo)
- Print-from-page CSS approach (less control, looks worse than a real PDF)
